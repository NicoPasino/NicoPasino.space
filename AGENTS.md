# AGENTS.md

## Stack
- ASP.NET Core 9.0 MVC + Web API, C#, MySQL 8.0.39 via Pomelo EF Core 9, Mapster for DTO mapping, `dotenv.net` for config.
- Solution `NicoPasino.sln` with 4 projects:
  - `NicoPasino` ‚Äî web app (`Microsoft.NET.Sdk.Web`, `net9.0`).
  - `NicoPasino.Core` ‚Äî DTOs, models, interfaces, mappers, custom errors, utils.
  - `NicoPasino.Infra` ‚Äî DbContexts + repositories.
  - `NicoPasino.Servicios` ‚Äî business logic services.

## Build & run
- `dotnet build` at repo root, or `dotnet run --project NicoPasino`.
- Local URLs come from `NicoPasino/Properties/launchSettings.json`:
  - `http` profile: `http://localhost:5230`.
  - `https` profile: `https://localhost:7267` + `http://localhost:5230`.
  - `NicoPasino.csproj.user` pins the active debug profile to `https`.
- `publish/` is a full self-contained Windows deploy output (AspNetCoreModuleV2 in-process via `web.config`, references `NicoPasino.exe`). Treat as build artifact; do not hand-edit or review diffs there.
- No test project, no `global.json`, no `.editorconfig`, no `.github/workflows/`, no CI, no lint/format/typecheck script. `dotnet build NicoPasino.sln` is the only verification.

## Environment / config
- `Program.cs` calls `DotEnv.Load()`. All real config comes from env vars.
- `NicoPasino/.env` is gitignored ‚Äî **must be created locally** before `dotnet run` will connect to any DB. Required keys:
  - `movies`, `ventas`, `notas` ‚Äî three MySQL connection strings, one per DbContext.
  - `ACCEPTED_ORIGINS` ‚Äî comma-separated CORS origins.
  - `MYSQL_VERSION` ‚Äî currently `8.0.39`. The same version is hard-coded in `Program.cs` as `new MySqlServerVersion(new Version(8, 0, 39))`; keep both in sync if you change it.
- `appsettings.json` contains a stale `MoviesConnectionString` (localdb) that is **not used** at runtime ‚Äî ignore it.

## Architecture
- Three DbContexts, one per area, in `NicoPasino.Infra/Data/`: `moviesdbContext`, `ventasdbContext`, `notasdbContext`. Entities live in `NicoPasino.Core/Modelos/{MoviesMySql,Ventas,Notas}`.
- Mapster is registered manually in `NicoPasino.Core/Mapper/MappingConfig.cs` and invoked from `Program.cs` (`MappingConfig.VentasMappings()`, `MappingConfig.NotasMappings()`). Add a new mapping there, then call it from `Program.cs`.
- Repository hierarchy in `NicoPasino.Infra/Repositorio`: `RepositorioGenericoBase<T>` is the CRUD base; `RepositorioGenericoMovies<T>`, `RepositorioGenericoVentas<T>`, `RepositorioGenericoNotes<T>` are context-specific subclasses. `UnitOfWorkMovie` is the movies-only Unit of Work.
- DI in `Program.cs:60-74` is **explicit, not open-generic**:
  - `IRepositorioGenerico<T>` ‚Üí `RepositorioGenericoMovies<T>` (movies only).
  - `IRepositorioGenericoVentas<T>` ‚Üí `RepositorioGenericoVentas<T>`.
  - `IRepositorioGenerico<Cards>` ‚Üí `RepositorioGenericoNotes<Cards>` ‚Äî only `Cards` is wired. To add another notas entity you must register a new closed-generic mapping here.
  - `IUnitOfWorkMovie` is registered but is not currently injected by `MovieServicio` (the service uses `IRepositorioGenerico<T>` directly). Don't rely on UoW without verifying a consumer first.

## Migrations / schema
- No `Migrations/` folder, no `Microsoft.EntityFrameworkCore.Design`, no `dotnet-ef` tool. Schema is managed out-of-band (manual SQL on the MySQL host or via EF Core Power Tools). The commented-out `Migrate()` block in `Program.cs` is a known gap, not a bug to remove.
- EF Core Power Tools is used to generate models. Configs: `NicoPasino.Core/efpt.config.json` and `NicoPasino.Infra/efpt.config.json` (both targeting the `notas` schema/`cards` table ‚Äî these configs are duplicated and only the notas path is wired up). Generated files are marked `<auto-generated>` ‚Äî do not hand-edit; regenerate.

## Conventions / quirks
- UI copy is in Spanish: error messages, `TempData` keys (`Msg`, `Tipo`), `ViewData` titles. Keep it that way.
- `Home/Index` is the portfolio landing page; sub-apps mount at `/Movies/Index` and `api/[controller]`. The `Movies` section is the only fully-Rendered MVC area; `Notes` and `Ventas` are API-only (see `NotesController`, `VentasController`).
- Status code re-execute middleware in `Program.cs:98` routes unmatched codes to `Home/NotFound` with `?statusCode={0}`. Several controllers instead call `NotFound(404, msg)` which is a different path ‚Äî fine, but don't mix them up.
- `VentasController` is split across partials in `NicoPasino/Controllers/`: `VentasController.cs` (class shell) + `Ventas.Productos.cs` + `Ventas.Clientes.cs` + `Ventas.Ventas.cs`. Add new ventas endpoints to the matching partial.
- Notas-specific quirks (`NicoPasino.Servicios/Servicios/Notas/NotasServicio.cs`):
  - `Cards.Id` is the DB primary key; `Cards.IdPublica` is the public-facing id. The DTO exposes a single `Id`, mapped via Mapster to `IdPublica`.
  - `GetById(int id)` looks up by `IdPublica`, not by DB Id.
  - `Create` **overwrites** the client-supplied `Id` with `Random.Next(1, 9999999)` to assign a new public id. Do not trust the client's value.
  - `Enable(int id, bool estado)` actually performs a hard `Delete`; the `estado` argument is ignored. Treat it as delete.
- `RepositorioGenericoBase.GetAsync` uses `AsNoTracking()`. If you need to update the entity returned from it, re-attach or fetch via `ListarAsync`/`Find` (which track).
- CORS allow-list in `.env` includes `https://0.0.0.0` and `http://0.0.0.0` ‚Äî keep them if you develop behind Docker port mapping; trim if you want a strict list.
- `Program.cs:23` has a commented-out `UseUrls("https://0.0.0.0:5000")` with a loud `TODO: DESACTIVARRRRRR‚Ä¶` marker. Leave it commented.

## Things that will bite you
- The four projects target `net9.0`, but the development machine currently has only the **.NET 10** SDK installed (`10.0.203`, runtime `10.0.7`). `dotnet build` / `dotnet run` will fail until a .NET 9 SDK is installed, or the `TargetFramework` is bumped to `net10.0` (re-test Mapster / Pomelo compatibility afterwards).
- The `NicoPasino/Core/Infra/Servicios/efpt.config.json.user` files and the per-project `bin/`, `obj/`, `wwwroot/imgs`, `wwwroot/notes`, `wwwroot/movie`, `wwwroot/codes` directories are intentionally gitignored (see `.gitignore`). Do not commit them; do not assume they are missing if a clone is fresh.
- `appsettings.json` is checked in but contains no secrets; secrets live only in the (gitignored) `.env`. Never paste a real connection string into a tracked file.
- `NicoPasino/NicoPasino.csproj.user` is local IDE state. Don't change it unless you mean to switch the default debug profile.
